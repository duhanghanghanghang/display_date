# 虚拟试衣 - 完善方案

## 一、现状分析

### 1.1 后端现状
- **实体**：`WardrobeOutfit` 含 `name`、`items`(Map<categoryId, itemId>)、`occasion`、`season`、`imageUrl`
- **API**：GET/POST/PATCH/DELETE `/wardrobe/outfits` 已完整
- **问题**：`items` 的 key 实际为 categoryId，与注释「部位」不一致；搭配返回无衣服详情，前端需二次查

### 1.2 前端现状
- **选择逻辑**：按分类选衣服，每分类最多一件，`selectedItems: { categoryId: itemId }`
- **展示区**：仅 emoji 🧍 + 选中数量，未展示选中衣服图片
- **保存弹窗**：只有名称，无 occasion、season
- **已保存搭配**：卡片只显示名称和件数，点击加载后无视觉反馈
- **Bug**：`allItems[category.id]` 可能 undefined 导致报错；删除搭配 `wx.showModal` 未正确 await

---

## 二、完善目标

1. **虚拟人物展示区**：按部位展示选中衣服的图片（上衣、裤子、外套等分层）
2. **保存/编辑**：支持填写 occasion、season
3. **已保存搭配**：卡片展示搭配预览图或衣服缩略图；支持编辑
4. **健壮性**：修复空数据、异步弹窗等 bug
5. **体验**：加载反馈、空状态、统一样式

---

## 三、详细方案

### 3.1 数据模型约定

| 字段 | 说明 |
|------|------|
| `items` | `{ categoryId: itemId }`，key 为分类 ID（代表部位如上衣、裤子），value 为衣服 ID |
| `occasion` | 适用场合，如：上班、休闲、约会 |
| `season` | 适用季节，如：春、夏、秋、冬 |

### 3.2 后端改动

| 项 | 说明 |
|----|------|
| **GET /wardrobe/outfits** | 返回搭配列表时，可选扩展：每个 outfit 的 items 中附带 `itemId -> { id, name, imageUrl, categoryName }`，减少前端请求 |
| **新增扩展响应** | 或保持现状，前端用已有 allItems 映射即可 |

**结论**：后端暂不改，前端用 `allItems` + `items` 映射即可获取衣服详情。

### 3.3 前端改动

#### A. 虚拟人物展示区
- 按分类顺序（或固定部位顺序：上衣→裤子→外套等）展示选中衣服
- 每个部位显示：分类名 + 衣服缩略图
- 无选中时显示占位（如「点击下方选择上衣」）
- 布局：上下或左右排列，适配小屏

#### B. 保存弹窗
- 名称（必填）
- 场合（可选，picker 或输入）
- 季节（可选，picker：春/夏/秋/冬）
- 保存时一并提交 occasion、season

#### C. 已保存搭配
- 卡片：展示名称 + 搭配内衣服的缩略图（2–3 张）或首图
- 点击：加载到选择区，并高亮已选
- 长按/编辑按钮：弹出编辑弹窗（可改名称、场合、季节、更换衣服）
- 删除：修复 wx.showModal 异步逻辑

#### D. Bug 修复
- `allItems[category.id]`：判空，`(allItems[category.id] || []).length`
- 请求衣服：outfit 页请求 `size=100` 以获取更多衣服
- 删除搭配：`wx.showModal` 使用 `success` 回调内的 `res.confirm` 判断

#### E. 空状态与加载
- 无分类时提示「请先添加分类和衣服」
- 加载中显示 loading
- 无已保存搭配时提示「保存你的第一套搭配」
